# PROJECT RULES

Never change architecture without approval.

Never replace technologies.

Always maintain TypeScript strict mode.

Always maintain Prisma ORM.

Always maintain PostgreSQL.

Always maintain Razorpay integration.

Never hardcode business data.

All content must come from database.

Every feature must be production grade.

Every form must use Zod validation.

Every API must have validation.

Every page must be SEO optimized.

Every feature must be mobile responsive.

Every change must update documentation.

No placeholder implementations.

No mock payment logic.

No temporary code in production.

No duplicated components.

No duplicated API routes.

Maintain enterprise-grade code quality.

Maintain clean architecture.

Think like a SaaS platform, not a landing page.
